# カバレッジを確定表の閾値まで満たす

> **呼び出し元**: `/implement-from-issue` の品質ゲート、または `/fix-implementation` のカバレッジ不足対応から呼ばれる補助 skill。**合格ラインの正典は `batch-00-stack.md` #11**（INSTRUCTION 100% / BRANCH 90%、除外後）。分岐は 90% 基準であり、到達不能分岐のための 100% 全域強制はしない。単独起動も可。

次の対象のカバレッジを改善する: $ARGUMENTS

## 指示

1. 最新のカバレッジレポート（`target/site/jacoco/jacoco.xml` または `target/site/jacoco/index.html`）を確認する。未生成でプロジェクト設定済みなら `mvn verify` を実行して生成する。
2. 未カバーの業務ロジック、分岐、異常系、入力検証を特定する（特にバッチ処理の Skip/Retry パス）。
3. 実質的なギャップを埋めるために必要最小限の意味あるテストを追加する。
4. 現状設計では妥当にテストできない場合のみリファクタを行う。
5. 再度 `mvn verify` を実行してカバレッジを計測し、`batch-00-stack.md` #11 の閾値達成を確認する。残ギャップを報告する。
6. 生成コードや非本質的なフレームワーク接着層（Spring Batch の自動生成 Bean 等）が閾値達成の障害なら、正当な最小除外として pom の jacoco excludes に 1 行理由コメント付きで追加する。分岐は 90% 基準であり、残る到達不能分岐のためだけのテストは書かない。
