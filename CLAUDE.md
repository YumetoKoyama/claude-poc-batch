# claude-poc-batch 向け指示（バッチ実装リポジトリ）

本リポジトリは配送マッチング Web アプリのバッチ処理実装を担う。設計書の正典は claude-poc-docs、親（横断ルール）は claude-poc-rules。

## 採択ゲート

- 実装の開始 = 人間が対象 Issue に `@claude` とコメントすること（Issue は採択済み設計書のマージを契機に自動起票される）。
- Claude・skill が自ら Issue 起票・コメント・マージして作業を開始してはならない。

## 実装ルール（ルール整備後に有効）

- バッチ設計の正典は claude-poc-docs の `docs/design/バッチ設計.md`（トランザクション単位・再実行・分割・監視連携・失敗時挙動）。
- 技術スタックの未確定項目を既定値で補完しない。中断して人間に指定を求める。
- 設計書の受け入れ条件（AC-XXX）を実装・テスト・PR で引用する。品質ゲート: 単体テスト・静的解析。E2E は実行しない。
- feature ブランチで作業し、PR 本文に `Closes #N` を記載する。

## 共通ルール

- 応答・コミットメッセージ・PR・作業ログは日本語で記述する。
- 本 CLAUDE.md と `.claude/rules/` は編集禁止（protect-canon フック。変更は人手で行う）。
